## 評分模型邏輯（v2）

模擬 FICO 五大因素，總分範圍 **300–850**：

```
總分 = clamp(使用率基礎分 + 還款紀錄 + 信用年齡 + 信用種類 + 新申請次數, 300, 850)
```

### 各因子計算方式

**① 使用率基礎分（30%）｜範圍 620–785**

| 使用率 | 基礎分 |
|---|---|
| 0% | 785 |
| ≤10% | 780 |
| ≤20% | 765 |
| ≤30% | 745 |
| ≤40% | 725 |
| ≤50% | 705 |
| ≤60% | 685 |
| ≤75% | 660 |
| ≤90% | 640 |
| ≤99% | 625 |
| >99% | 620 |

**② 還款紀錄（35%）｜最多扣 193 分**

| 事件 | 扣分 |
|---|---|
| 30天以上逾期（每次） | -38 |
| 60天以上逾期（每次） | -38（累加） |
| 公開不良紀錄 | -25 |
| 催收帳戶 | -19 |
| 31–120天遲繳 | -38 |
| 16–30天遲繳 | -13 |

**③ 信用年齡（15%）｜-63 到 +20**

| 年齡 | 分數 |
|---|---|
| < 1年 | -63 |
| 1–3年 | -35 |
| 3–7年 | -15 |
| 7–15年 | 0 |
| > 15年 | +20 |

**④ 信用種類（10%）｜-25 到 +30**

- DTI < 15% → +30
- 債務整合且 DTI ≥ 36% → -25（高風險覆蓋）
- 其他 → +15

**⑤ 新申請次數（10%）｜-39 到 +15**

| 次數 | 分數 |
|---|---|
| 0次 | +15 |
| 1次 | 0 |
| 2–3次 | -15 |
| 4–5次 | -27 |
| ≥6次 | -39 |

## 客群分類邏輯

| 客群 | 條件 | 建議行動方向 |
|---|---|---|
| 違約 | loan_status = Charged Off / Default | 轉法務/委外催收 |
| 高風險 | total < 580 或 年收入 < 循環餘額 | 整合型貸款方案 |
| 優質 | total ≥ 700 且 年收入 > 循環餘額 | 理財商品/備用信貸 |
| 正常 | 其他 | 觀察 |

## v1 → v2 校準說明

v1 版本存在一個問題：**年收入高、DTI 低的客戶，若持有大額循環餘額，仍被錯誤判定為高風險**。

**校準案例**：客戶 ID 55140826
- 年收入：890 萬（USD）
- DTI：0.09（極低）
- 使用率：102%（因循環餘額略超額度）
- v1 評分：450（高風險）❌
- v2 評分：635（正常）✅

**校準方向**：
1. 使用率查表從線性改為分段式，避免小幅超額造成斷崖式扣分
2. 信用種類改為以 DTI 判斷，而非單純計算產品種數
3. 加入 `override` 機制：高 DTI 債務整合客戶直接歸類高風險

## 前端功能說明

### 信用評分模擬器（simulator_v4.html）

**Step 1｜填資料**
- 五大因子逐一填寫
- 多張信用卡支援、逐卡使用率視覺化

**Step 2｜看評分**
- 即時計算 FICO 模擬分數（300–850）
- 評分量尺、各因子貢獻圖
- 個人化改善建議

**Step 3｜採取行動**
- 還款試算：雪崩法 vs 雪球法比較
- 信貸試算：依評分推薦利率範圍、DTI 壓力測試

### 匯入資料庫（DB彈窗）
- 四大客群分布（優質 36.2% / 高風險 3.5% / 違約 6.2% / 正常 54%）
- TOP100 即時載入（API 回應 < 0.5 秒）
- 客戶明細：評分組成、五因子視覺化、轉貸試算、建議行動

### 分析報告
- **P15**：使用率極高月份（>90%）違約率從 26.3% 升至 34.1%，非極高月份維持 ~21%（台灣資料）
- **P16**：甜甜圈圖（風險等級分布）、信用等級 A-G 違約率（美國資料）
- **P17**：各貸款用途違約筆數 Top8，債務整合 27,599 筆最高（美國資料）
- **P19**：真實違約率（3.8%→6.6%）vs 基數效應誤判陷阱（美國資料）
